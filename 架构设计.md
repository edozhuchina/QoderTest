# 架构设计

## 1. 技术栈总览

| 层级 | 技术选型 | 版本/说明 |
|------|----------|-----------|
| 前端 | HTML + CSS + JavaScript | 原生三件套，无框架依赖 |
| 后端框架 | Django (LTS) | 5.2（当前最新长维护版本） |
| REST API | Django REST Framework (DRF) | 3.15+ |
| 认证方案 | djangorestframework-simplejwt | JWT 前后端分离标配 |
| 跨域处理 | django-cors-headers | 允许前端跨域请求 |
| 序列化/校验 | DRF Serializer（原生） | ModelSerializer / Serializer |
| 数据库 | SQLite（开发）/ PostgreSQL（生产） | Django ORM |

```

 │                             │
 │  POST /api/auth/login/      │
 │  { username, password }     │

 │                             │  验证凭据

 │←────────────────────────────│
 │                             │
 │  GET /api/resource/         │

 │  200 { data }               │
 │←────────────────────────────│
 │                             │

 │←────────────────────────────│
```

### 4.4 Serializer 设计规范

```python
# apps/users/serializers.py
from rest_framework import serializers
from django.contrib.auth.models import User

class UserSerializer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = ['id', 'username', 'email', 'date_joined']
        read_only_fields = ['id', 'date_joined']

class UserCreateSerializer(serializers.ModelSerializer):
    password = serializers.CharField(write_only=True, min_length=8)
    password_confirm = serializers.CharField(write_only=True)

    class Meta:
        model = User
        fields = ['username', 'email', 'password', 'password_confirm']

    def validate(self, attrs):
        if attrs['password'] != attrs['password_confirm']:
            raise serializers.ValidationError("两次密码不一致")
        return attrs

    def create(self, validated_data):
        validated_data.pop('password_confirm')
        return User.objects.create_user(**validated_data)
```

## 5. 前端架构设计

### 5.1 API 请求封装（js/api.js）

```javascript
const API_BASE = 'http://127.0.0.1:8000/api';

class ApiClient {
    constructor() {
        this.accessToken = localStorage.getItem('access_token') || null;
    }

    setToken(access) {
        this.accessToken = access;
        localStorage.setItem('access_token', access);
    }

    clearToken() {
        this.accessToken = null;
        localStorage.removeItem('access_token');
        localStorage.removeItem('refresh_token');
    }

    async request(endpoint, options = {}) {
        const headers = {
            'Content-Type': 'application/json',
            ...options.headers,
        };
        if (this.accessToken) {
            headers['Authorization'] = `Bearer ${this.accessToken}`;
        }

        let response = await fetch(`${API_BASE}${endpoint}`, {
            ...options,
            headers,
        });

        // Access Token 过期，尝试刷新
        if (response.status === 401) {
            const refreshed = await this.refreshToken();
            if (refreshed) {
                headers['Authorization'] = `Bearer ${this.accessToken}`;
                response = await fetch(`${API_BASE}${endpoint}`, {
                    ...options,
                    headers,
                });
            } else {
                this.clearToken();
                window.location.href = '/login.html';
            }
        }

        return response;
    }

    async refreshToken() {
        const refresh = localStorage.getItem('refresh_token');
        if (!refresh) return false;

        const res = await fetch(`${API_BASE}/auth/refresh/`, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ refresh }),
        });

        if (res.ok) {
            const data = await res.json();
            this.setToken(data.access);
            return true;
        }
        return false;
    }
}

const api = new ApiClient();
```

### 5.2 认证逻辑（js/auth.js）

```javascript
async function login(username, password) {
    const res = await fetch(`${API_BASE}/auth/login/`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username, password }),
    });
    if (res.ok) {
        const { access, refresh } = await res.json();
        api.setToken(access);
        localStorage.setItem('refresh_token', refresh);
        return true;
    }
    return false;
}

async function register(username, email, password, passwordConfirm) {
    const res = await fetch(`${API_BASE}/users/register/`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            username, email, password, password_confirm: passwordConfirm,
        }),
    });
    return res.ok;
}

function logout() {
    api.clearToken();
    window.location.href = '/login.html';
}
```

## 6. 跨域策略

| 配置项 | 开发环境值 | 说明 |
|--------|-----------|------|
| `CORS_ALLOWED_ORIGINS` | `http://localhost:5500` | 前端开发服务器地址 |
| `CORS_ALLOW_CREDENTIALS` | `True` | 允许携带认证信息 |
| `CORS_ALLOW_METHODS` | 默认全部 | GET/POST/PUT/PATCH/DELETE |
| `CORS_ALLOW_HEADERS` | 默认含 `Authorization` | JWT Header 放行 |

> **注意：** 生产部署时，前端打包为 Django 静态文件提供服务，不再需要 CORS 配置。

## 9. 安全规范

| 规范 | 实现方式 |
|------|---------|
| 密码不明文传输 | HTTPS（生产环境） |
| 密码不明文存储 | Django `make_password` / `create_user` |
| Token 安全存储 | 前端 `localStorage`（开发阶段），生产建议 `httpOnly Cookie` |
| CSRF 防护 | 前后端分离模式下 JWT 认证无需 CSRF |
| SQL 注入防护 | Django ORM 参数化查询 |
| XSS 防护 | DRF 默认转义输出 |
